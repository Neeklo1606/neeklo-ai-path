# neeklo.ru — сопоставление чеклиста с проектом

**Дата:** апрель 2026  
**Кодовая база:** основной продукт — папка `neeklo-ai-path` (React + TypeScript, Vite; админка и публичный сайт; API на Node/Prisma в `server/`).

**Легенда статусов**

| Символ | Значение |
|--------|----------|
| ✅ | Уже реализовано или заложено в коде |
| ⚠️ | Частично (нужна доработка или контент) |
| ❌ | В репозитории не найдено / несовпадает с ТЗ |
| 🔧 | В основном контент, медиа или настройки (CMS), не только код |
| 📱 | Требуется проверка на реальном устройстве (вне репозитория) |
| 🔄 | Зависит от внешнего процесса (Lovable, хостинг, Telegram и т.д.) |

## 1. Архитектура проекта (кратко)

- **Фронт:** `src/` — маршруты в `App.tsx` (`/`, `/chat`, `/services`, `/works`, `/projects`, `/profile`, `/admin`, …).
- **Контент страниц:** CMS (`cmsPageBySlug`, блоки на главной в `CmsHomePage.tsx`), услуги и работы — страницы `services` / `works` из CMS.
- **Брендинг:** логотипы идут из **настроек** (`public.brand.logo_url` и др.) через `BrandLogo.tsx`, не из статического `МАСКОТ.png` в `public/` (в репозитории в `public/` сейчас почти нет картинок — только `placeholder.svg`, `robots.txt`).
- **Нижняя навигация:** `BottomNav.tsx` — уже **Главная / Чат / Проекты / Профиль**, активный таб, `safe-area-inset-bottom`, `backdrop-filter` на баре.

Это важно для планирования: часть пунктов чеклиста закрывается **загрузкой файлов в админку и заполнением CMS**, а не только правками React.

| Задача | Сопоставление с проектом | Что нужно для реализации | Что можно «подкрутить» без полного редизайна |

| Favicon | ⚠️ Обычно `index.html` / Vite `public/` — нужно сверить с деплоем | Добавить `favicon.ico` / `apple-touch-icon` в `public/` и ссылки в `index.html` | Один набор иконок из нового логотипа |
| Маскот на главной | ✅ Hero в `CmsHomePage` — слот **mascot** из CMS блоков | В CMS для главной указать медиа **mascot** (новый шар с шапкой и очками) | Убрать дублирование с legacy `imageId` через `cms-block-images` уже учтено |

| Убрать старого робота | 🔧 Поиск по репо: «робот» как **персонаж** не зафиксирован в коде явно — скорее **старые картинки в CMS/медиа** | Заменить ассеты в админке и в блоках страниц | Проверить кэш CDN/браузера после смены URL |

## 3. Мобильная версия

| Задача | Сопоставление | Реализация | Быстрые улучшения |
|--------|---------------|------------|-------------------|
| Проверка на iPhone | 📱 | — | Чеклист ручной |
| `once: true` на анимациях | ⚠️ Framer Motion в `CmsHomePage` и др.; `fadeUp` / `whileInView` — **нужно сверить** каждый вызов | Пройти `grep` по `whileInView` и `viewport` | Добавить `viewport={{ once: true }}` где его нет |
| Bottom nav фиксирован | ✅ `fixed bottom-0`, 4 пункта как в ТЗ | — | При необходимости высота/z-index |
| Убрать backdrop-blur | ⚠️ Есть в `BottomNav` и модалках | Снизить blur или заменить на непрозрачный фон на `sm:hidden` | Только на мобильной ветке |
| `will-change: transform` | ⚠️ У маскота на главной уже есть в inline-стиле | Распространить на карточки с `hover:-translate` | Точечно, чтобы не раздувать слои |
| Отключить scroll-анимации &lt;768px | ❌ Как единая политика не видно | `useMediaQuery` или `MotionConfig` + условные `initial`/`animate` | Флаг в одном месте |
| `overflow-x: hidden` | ⚠️ Частично (`ServicesPage` есть) | Глобально: `body` + layout в `App.tsx` / `index.css` | Проверить секции с широкими детьми |
| `padding-bottom: env(safe-area)` | ⚠️ Есть в `BottomNav`; «везде» — не гарантировано | Пройти страницы с фиксированным низом/полноэкранным чатом | Единый `pb-[calc(...+env())]` для layout |
| Шрифт не меньше 13px | ⚠️ В `BottomNav` подписи **10px** — **разошлось с ТЗ** | Поднять минимум до 13px на мобильных | Точечно в nav и мелком тексте |
| Touch targets ≥44×44 | ⚠️ Не везде явно | Аудит кнопок и `min-h-[44px] min-w-[44px]` | Иконки + padding |

## 18. Приоритизация (рекомендация без кода)

**P0 — блокеры продукта и доверия**

- Исправление бага «услуга со второго раза».
- Согласование бренда: загрузка логотипа/маскота/favicon через понятный путь (админка + проверка кэша).
- Стабильность чата и отсутствие «белого экрана» при ошибках загрузки.

**P1 — конверсия и UX**

- Модалы услуг с CTA в `/chat`, фильтры категорий на `/services`.
- Единые модалы карточек (главная, работы, услуги).
- Профиль: защита маршрута и редирект после логина.

**P2 — мобильная полировка**

- `once: true`, отключение тяжёлых анимаций на мобильных, шрифт nav ≥13px, touch targets.
- Уменьшение `backdrop-blur` на слабых устройствах.

**P3 — контент и интеграции**

- 15 работ, отзывы, Авито, webhook Avito, уведомления менеджеру.
- Поиск по услугам.

**P4 — операционка**

- Ручной прогон на iPhone, Chrome/Safari, Lovable sync.
